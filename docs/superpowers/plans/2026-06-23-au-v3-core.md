# AU v3 Core (Plan A) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** host **AU v3 (`AUAudioUnit`)** plugins' audio natively on macOS — async instantiation, the
`renderBlock` audio path, the `AUParameterTree`, and `fullState` — as a second plugin class
(`AudioUnitV3Plugin`) inside the existing `MidiSharp.Hosting.AudioUnit` adapter. The load-bearing risk is the
**Objective-C block ABI** (construct + invoke), de-risked by **Task 0** before any adapter code.

**Architecture:** discovery is unchanged and already v3-aware (`AudioComponentFindNext`); `AudioUnitFormat.Load`
gains a branch — components flagged `kAudioComponentFlag_IsV3AudioUnit` route to `AudioUnitV3Plugin`, others keep
the shipped v2 `AudioUnitPlugin`. `AUAudioUnit` is block-centric: the host **invokes** the AU's `renderBlock` and
**constructs** the `pullInputBlock` + completion handlers it hands the AU. Audio is non-interleaved float32 (an
`AVAudioFormat` standard format), mapping straight onto the planar bus — same push↔pull shim shape as v2, but the
call target is an Obj-C block.

**Spec:** `docs/superpowers/specs/2026-06-23-au-v3-hosting-design.md` (§5.1–5.4, §6, §9, §12). **Task 0 (the
block-ABI spike) gates the rest.**

**Verification:** no v3 AU is installed, but `AUAudioUnit` wraps v2 components (`AUAudioUnitV2Bridge`), so the
whole path is verified against **`AULowpass` wrapped as an `AUAudioUnit`**. v3-only edges (async-*required*
instantiation, `LoadOutOfProcess`) self-skip.

**Tech Stack:** C# (net10.0), AudioToolbox/AVFoundation/AppKit `objc_msgSend` + hand-built Obj-C blocks, xUnit.
**arm64, macOS-only.**

## Global Constraints

- **`dotnet`**, net10.0, **0/0**, dependency-free (system frameworks only), **additive** — the shipped v2 path
  (`AudioUnitPlugin`, discovery, editor fix) is untouched; v3 is reached only for v3 components on macOS.
- **`renderBlock` invocation is realtime-clean** — no managed alloc, no locks; the `pullInputBlock` is
  `[UnmanagedCallersOnly]`, its context a pinned `GCHandle`; blocks built once in Activate.
- macOS-only tests self-skip off-OS; the v2-wrapped checks never skip on macOS.

## Task 2 — Format routing + async instantiation

- [ ] `AudioUnitFormat.Load`: read the component's `kAudioComponentFlag_IsV3AudioUnit`; v3 →

- [ ] `AudioUnitV3Plugin.Activate`: async-instantiate the `AUAudioUnit` (completion block + managed reset event,
      awaited off the audio thread); set bus formats, `maximumFramesToRender`, `pullInputBlock` (effects),
      cache `renderBlock`, `allocateRenderResourcesAndReturnError:`. Dispose: `deallocateRenderResources` +
      release the AU/blocks/`GCHandle`s.
- [ ] **Test** `AudioUnitV3Tests.Instantiates_aulowpass_as_auaudiounit` (macOS-only): loads and activates. Commit.

## Task 3 — Render path

- [ ] `Process`: stash input channels; build the output `AudioBufferList`; `InvokeRender(renderBlock, flags, ts,

- [ ] **Test**: render a 200 Hz tone (passes) and 18 kHz (attenuated) through `AULowpass`-as-`AUAudioUnit` —
