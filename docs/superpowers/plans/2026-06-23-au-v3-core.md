# AU v3 Core (Plan A) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** host **AU v3 (`AUAudioUnit`)** plugins' audio natively on macOS — async instantiation, the
`renderBlock` audio path, the `AUParameterTree`, and `fullState` — as a second plugin class
(`AudioUnitV3Plugin`) inside the existing `MidiSharp.Hosting.AudioUnit` adapter. The load-bearing risk is the
**Objective-C block ABI** (construct + invoke), de-risked by **Task 0** before any adapter code.

**Architecture:** discovery is unchanged and already v3-aware (`AudioComponentFindNext`); `AudioUnitFormat.Load`
gains a branch — components flagged `kAudioComponentFlag_IsV3AudioUnit` route to `AudioUnitV3Plugin`, others keep
the shipped v2 `AudioUnitPlugin`. `AUAudioUnit` is block-centric: the host **invokes** the AU's `renderBlock` and
**constructs** the `pullInputBlock` + completion handlers it hands the AU. Audio is non-interleaved float32 (an
`AVAudioFormat` standard format), mapping straight onto the planar bus — same push↔pull shim shape as v2, but the
call target is an Obj-C block.

**Spec:** `docs/superpowers/specs/2026-06-23-au-v3-hosting-design.md` (§5.1–5.4, §6, §9, §12). **Task 0 (the
block-ABI spike) gates the rest.**

**Verification:** four **real v3 effects** are installed (`DimChorus`/`FakeTake`/`LevLr`/`J_NO Chorus`) — all
`RequiresAsync` + `CanLoadInProcess=false`, so the genuine **async + out-of-process** path is the primary one and
is verified directly against them. A v2-wrapped `AULowpass`-as-`AUAudioUnit` adds a fast in-process API sanity
check. (The OOP-only nature of the installed units corrected an earlier "OOP is optional" assumption.)

**Tech Stack:** C# (net10.0), AudioToolbox/AVFoundation/AppKit `objc_msgSend` + hand-built Obj-C blocks, xUnit.
**arm64, macOS-only.**

## Global Constraints

- **`dotnet`**, net10.0, **0/0**, dependency-free (system frameworks only), **additive** — the shipped v2 path
  (`AudioUnitPlugin`, discovery, editor fix) is untouched; v3 is reached only for v3 components on macOS.
- **`renderBlock` invocation is realtime-clean** — no managed alloc, no locks; the `pullInputBlock` is
  `[UnmanagedCallersOnly]`, its context a pinned `GCHandle`; blocks built once in Activate.
- macOS-only tests self-skip off-OS; the v2-wrapped checks never skip on macOS.

## Task 0 — Block-ABI + async/OOP spike (de-risk the load-bearing design)  — **gate**

- [ ] Throwaway console (net10.0, arm64): construct an Obj-C **completion block** and call
      `[AUAudioUnit instantiateWithComponentDescription:options:completionHandler:]` for **`DimChorus`**
      (`aufx:DIMC:MU4S`, a real v3 effect, options `LoadOutOfProcess` since `CanLoadInProcess` is clear); block
      a managed reset event until the handler delivers the `AUAudioUnit`. Also do the in-process v2-wrapped
      `AULowpass` variant as a faster sanity path.
- [ ] Set a stereo non-interleaved-float `AVAudioFormat` on the output (+input) bus; set `maximumFramesToRender`;
      construct a **`pullInputBlock`** that fills the requested `AudioBufferList` from a known input (a sine);
      `allocateRenderResourcesAndReturnError:`.
- [ ] Read the `renderBlock` property, **invoke** it (read `invoke` at block offset 16, call) over a 512-frame
      block, and assert the output **differs from** the input (the chorus acts — proves the OOP render round-trips
      through XPC). **Measure the per-block latency** of the OOP render (does it fit the lock-step budget?).
- [ ] **Confirm and record** the block layout (`invoke` @16, captured `context` @32), the
      `_NSConcreteGlobalBlock`/`_NSConcreteStackBlock` symbol, and the `AudioComponentInstantiationOptions`
      values; confirm `parameterTree` + `fullState` on `DimChorus`. **Record SPIKE PASS (with the OOP latency and
      any corrected offsets/values) before Task 1.** If OOP render latency is unworkable, flag it here — it
      gates the whole design.

## Task 1 — Block bridge + framework load (`AuBlocks.cs`)

- [ ] `AuBlocks`: `InvokeRender(block, …)` (read `*(void**)(block+16)`, call via `delegate* unmanaged[Cdecl]`);
      `MakeBlock(invokeFnPtr, contextGCHandle)` (build `{ isa=_NSConcreteGlobalBlock; flags; reserved;
      invoke; descriptor=&{0,size}; context }`); helpers to read a vended block property and free a constructed
      block. Load AVFoundation so `AUAudioUnit`/`AVAudioFormat` resolve.
- [ ] Builds 0/0; uses the spike-verified offsets/symbols.

## Task 2 — Format routing + async instantiation

- [ ] `AudioUnitFormat.Load`: read the component's `kAudioComponentFlag_IsV3AudioUnit`; v3 →
      `new AudioUnitV3Plugin(...)`, else the existing `AudioUnitPlugin`. (Discovery unchanged.)
- [ ] `AudioUnitV3Plugin.Activate`: async-instantiate the `AUAudioUnit` (completion block + managed reset event,
      awaited off the audio thread); set bus formats, `maximumFramesToRender`, `pullInputBlock` (effects),
      cache `renderBlock`, `allocateRenderResourcesAndReturnError:`. Dispose: `deallocateRenderResources` +
      release the AU/blocks/`GCHandle`s.
- [ ] **Test** `AudioUnitV3Tests.Instantiates_dimchorus_v3` (macOS-only): the real v3 `DimChorus` async-loads
      out-of-process and activates; an in-process v2-wrapped `AULowpass` arm covers the sync path. Commit.

## Task 3 — Render path

- [ ] `Process`: stash input channels; build the output `AudioBufferList`; `InvokeRender(renderBlock, flags, ts,
      frames, 0, &abl, null, pullInputBlock)`; the `pullInputBlock` serves the stashed input; mirror mono→stereo.
- [ ] **Test**: render a tone through **`DimChorus`** and assert the output is non-silent and **differs from the
      input** (the chorus acts — proves the OOP render round-trips); the v2-wrapped `AULowpass` arm asserts
      filtering. Commit.

## Task 4 — Parameters (`AUParameterTree`)

- [ ] Build `PluginParameter[]` from `parameterTree.allParameters` (keyed by `AUParameterAddress`);
      `GetParameter` reads `param.value`, `SetParameter` writes it (off-thread).
- [ ] **Test**: changing a `DimChorus` parameter (e.g. depth/rate) changes the rendered output. Commit.

## Task 5 — State (`fullState`)

- [ ] `SaveState`: serialize the `fullState` `NSDictionary` via `CoreFoundation.ToData`; `LoadState`: bytes →
      `CreatePropertyList` → set `fullState`. Release CF objects.
- [ ] **Test**: set a param, `SaveState`, change it, `LoadState`, assert restored. Commit.
