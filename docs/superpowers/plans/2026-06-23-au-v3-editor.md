# AU v3 Editor (Plan C) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** open an **AU v3 plugin's native editor** through the existing macOS `EditorHost`, by implementing
`IPluginGui` on `AudioUnitV3Plugin` over `requestViewControllerWithCompletionHandler:`. Verified against an AU's
view controller (e.g. `AULowpass`/Surge wrapped as an `AUAudioUnit`), with the v2 path's `AUGenericView` as the
fallback.

**Architecture:** the Cocoa `EditorHost` backend, `EditorSession`, and the AU **neutral-background** fix all
exist and are **unchanged**. The only new surface is v3 view vending: `[au requestViewControllerWith
CompletionHandler:]` takes a constructed completion block; on the **main thread** it delivers an
`NSViewController` whose `.view` is the editor `NSView`. `IPluginGui.Create` requests it (awaiting the handler),
`SetParent` adds that view under the host content `NSView` (reusing `AuAppKit.AddSubview` +
`SetNeutralWindowBackground`). Editor calls are main-thread-only, so verification is the main-thread
`MacEditorHarness`, not xUnit.

**Spec:** `docs/superpowers/specs/2026-06-23-au-v3-hosting-design.md` (§5.6, §6, §10). **Prerequisite: Plan A**
(a loadable `AudioUnitV3Plugin`). The Cocoa `EditorHost` + `AuAppKit` are already shipped.

**Tech Stack:** C# (net10.0), AudioToolbox/AppKit `objc_msgSend` + a completion block (`AuBlocks`), xUnit for the
AppKit-free surface. **arm64, macOS-only.**

## Global Constraints

- Same as Plan A: `dotnet`, net10.0, **0/0**, dependency-free, additive. **No change to `EditorSession` or the
  Cocoa backend** — the v3 editor must ride them unchanged (as the v2 editor does), proving the seam.
- **Main thread for all editor calls** (the harness `Main` is thread 0; the worker is too).

## Task 1 — Request the v3 view controller

- [ ] Add to `AuBlocks`/`AudioUnitV3Plugin`: a completion block for `requestViewControllerWith
      CompletionHandler:`. `TryRequestView()` → call it, await the handler (managed reset event pumped on the
      main thread), read `viewController.view` (`NSView*`), retain it. Null when the AU vends no controller.

## Task 2 — Generic fallback

- [ ] When `TryRequestView` yields nothing, reuse the v2 path's `AUGenericView` over the underlying `AudioUnit`
      (the `AUAudioUnit` exposes the v2 `audioUnit` handle, or build the generic view from the component) — so a
      v3 AU without a custom controller still gets a usable editor.

## Task 3 — `IPluginGui` on `AudioUnitV3Plugin`

- [ ] Implement `IPluginGui` (mirroring the v2 `AudioUnitPlugin`): `Gui => this`; `HasEditor => true`;
      `IsApiSupported("cocoa", embedded)`; `Create` obtains the view (controller → generic) and retains it;
      `SetParent("cocoa", contentView)` adds it as a subview and sets the neutral window background;
      `TryGetSize` from the view frame; `Show`/`Hide`/`Destroy` toggle hidden / remove / release;
      `BindRunLoop`/`SetScale` no-ops.

## Task 4 — Verify embedding & acceptance

- [ ] Extend `MacEditorHarness` with an `EmbedAuV3` arm: route a v3 (or v2-wrapped) AU through
      `AudioUnitV3Plugin`, open an `EditorSession` on the main thread, assert `EmbeddedChildCount ≥ 1`. Plus an
      AppKit-free xUnit test for the editor surface (`Gui`/`HasEditor`/`IsApiSupported`).
- [ ] **Acceptance gate (spec §10, Plan C):** an AU's v3 view controller embeds a child `NSView` through the
      **unchanged** `EditorSession` (`PASS AU-v3`); generic fallback covers a controller-less AU. Solution
      **0/0**; the v2 editor path and Cocoa backend behaviorally unchanged.
- [ ] Update `docs/plugin-hosting-plan.md` and `CHANGELOG.md`. Commit. **Do not merge/push unless asked.**

## Notes

- This slice touches the AU adapter + harness **only**; needing any change to `EditorSession`/the Cocoa backend
  signals the seam is leaking — reconsider, don't patch around.
- Out-of-process (`LoadOutOfProcess`) editors — where the view comes from Apple's AU extension host over XPC —
  remain opt-in/out of scope (spec §11).
